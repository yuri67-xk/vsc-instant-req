# コントリビューションガイドライン

このリポジトリへコントリビュートしていただきありがとうございます。以下のガイドラインに従ってください。

## 前提条件

- Node.js 18.x以上
- VSCode 1.80.0以上
- Gitの基本的な知識

## 開発環境のセットアップ

1. リポジトリをフォーク
2. ローカルにクローン
```bash
git clone https://github.com/your-username/vsc-instant-req.git
cd vsc-instant-req
```

3. 依存関係をインストール
```bash
npm install
```

4. 開発ブランチを作成
```bash
git checkout -b feature/your-feature-name
```

## 開発ルール

### 1. コーディング規約
- TypeScriptで実装
- ESLintルールに準拠
- 有用なコメントを追加
- 変数名や関数名は意味のある名前を使用

### 2. コミットメッセージ規約
```
<type>(<scope>): <subject>

<body>

<footer>
```

**タイプ例:**
- feat: 新機能
- fix: バグ修正
- docs: ドキュメント更新
- style: コードフォーマット
- refactor: リファクタリング
- test: テスト追加
- chore: ビルドや補助ツールの変更

### 3. プルリクエスト
- プルリクエストは1つの変更にフォーカス
- タイトルは明確で具体的に
- 変更内容を説明
- 関連Issueがあれば参照
- テストを追加（該当する場合）

### 4. ブランチ戦略
- main: 安定版
- develop: 開発版
- feature/*: 新機能
- fix/*: バグ修正
- hotfix/*: 緊急修正

## レビュープロセス

1. プルリクエストを作成
2. 自動CI/CDチェックがパスすることを確認
3. 2人以上のメンテナーによるレビューを取得
4. 全てのチェックがパスしたらマージ

## セキュリティ

- セキュリティ関連の変更は、プルリクエストを作成する前に [SECURITY.md](./SECURITY.md) に従って報告
- 依存関係の追加はセキュリティレビュー対象

## リリースプロセス

1. versionをpackage.jsonで更新
2. CHANGELOG.mdを更新
3. タグを作成: `git tag v1.x.x`
4. GitHubでリリースを作成
5. VSIXをパッケージングしてマーケットプレイスに公開

## 行動規範

- 尊敬あるコミュニケーション
- 建設的なフィードバック
- 包摂的な言動
- ハラスメントの禁止

## ヘルプ

質問や不明点がある場合は:
- GitHub Issueを作成
- 既存のIssueを検索して重複を避ける
- 詳細な情報を提供
