# 使用例

## シナリオ1: ユーザー認証機能の追加

### 入力

**機能要件:**
- ユーザー登録・ログイン機能の実装
- JWT認証の導入
- パスワードのハッシュ化（bcrypt使用）

**エージェント選択:**
- 仕様設計書作成: System Architect
- 実装担当: Backend Developer

### 生成されるプロンプト

```
1. 以下の要件に基づいて機能改修したい。
   - ユーザー登録・ログイン機能の実装
   - JWT認証の導入
   - パスワードのハッシュ化（bcrypt使用）

2. 上記の要件をPlanningすること。

3. ユーザーがPlanningを承認したら、architectにdocs/specs/配下に仕様書を作成させる（specs/配下に分類可能な既存フォルダがあるかを確認し、ない場合は新規作成・ある場合はフォルダへ保存）

4. ドキュメント保存後、その仕様設計書に基づいて、backend-devに実装を行わせる。

5. 実装完了後はユーザーがテストするので、Git Commitを自動で行わないこと。
```

## カスタムエージェントの追加例

`settings.json`に独自のエージェントを追加する場合:

```json
{
  "agents": {
    "specWriters": [
      {
        "id": "api-designer",
        "name": "API Designer",
        "description": "RESTful API設計専門エージェント"
      },
      {
        "id": "database-architect",
        "name": "Database Architect",
        "description": "データベース設計専門エージェント"
      }
    ],
    "implementers": [
      {
        "id": "devops-engineer",
        "name": "DevOps Engineer",
        "description": "CI/CD・インフラ構築専門"
      },
      {
        "id": "security-expert",
        "name": "Security Expert",
        "description": "セキュリティ実装専門"
      }
    ]
  }
}
```

これにより、ドロップダウンに新しいエージェントが表示されるようになります。

## ワークフロー

1. **要件入力**: 実装したい機能を箇条書きで入力
2. **エージェント選択**: タスクに最適なエージェントを選択
3. **プロンプト生成**: 「プロンプト作成」ボタンをクリック
4. **コピー**: 「COPY」ボタンでクリップボードにコピー
5. **Claude Codeで実行**: コピーしたプロンプトをClaude Codeに貼り付けて実行

これにより、Plan Modeを活用した構造化されたワークフローが自動的に実行されます。
